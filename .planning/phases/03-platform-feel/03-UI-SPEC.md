## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — vanilla HTML/CSS/JS |
| Icon library | none — emoji icons only (existing pattern) |
| Font | Press Start 2P, weight 400 only, via Google Fonts CDN |

**Font loading rule (D-01):** Add exactly one `<link>` tag in `<head>` of `index.html` before `style.css`:

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Press+Start+2P&display=swap" rel="stylesheet">
```

Apply globally in `style.css`:

```css
body {
  font-family: 'Press Start 2P', monospace;
}
```

**Weight fallback note:** Press Start 2P ships only weight 400. All existing `font-weight: 700` declarations remain in CSS but render as 400. Do not remove them — they are harmless and remain valid for future font swaps.

## Typography

Press Start 2P is a bitmap/pixel font — it does not support fractional sizes or optical kerning. All sizes declared below are the exact pixel values already present in the codebase. The font swap does not require changing any size declarations.

| Role | Size | Weight | Line Height | Elements |
|------|------|--------|-------------|----------|
| Label / small | 12px | 400 | 1.4 | `.stat-label`, `.stat-sub`, `.standings-header`, `.standing-lap`, `.dice-result` |
| Body | 14px | 400 | 1.5 | `.back-link`, `.callout`, `.shop-current` |
| UI / interactive | 16px | 400 | 1.3 | `.standing-label`, `.position-badge`, `.result-row`, `.shop-maxed` |

**Critical sizing caveat:** Press Start 2P is wider and taller than system-ui at the same px value. After applying the font, visually verify that:
1. The header (`#top-bar`) does not overflow or wrap on 375px at 20px heading size — reduce `.game-title` to `16px` on mobile if it wraps.
2. The standings panel label text at 12px remains legible (not smaller than 10px effective rendering size).
3. Shop card headings at 18px remain within card bounds on 375px — reduce to 14px on mobile if needed.

These size adjustments are at executor discretion within the constraint that no text drops below 12px.

## Car Image Contract (D-02)

This is the primary new visual contract for Phase 3.

### Image Sources

| Car | ID | Image path | Represents |
|-----|----|-----------|------------|
| Player | `player` | `images/car-red.png` | Þú |
| AI 1 | `ai1` | `images/car-blue.png` | A1 |
| AI 2 | `ai2` | `images/car-green.png` | A2 |
| AI 3 | `ai3` | `images/car-yellow.png` | A3 |

### CSS Class: `.car-img`

Replace `.car-circle` rules with:

```css
.car-img {
  width: 80%;
  height: 80%;
  object-fit: contain;
  display: block;
  image-rendering: pixelated;
}

.car-img.car-player {
  outline: 2px solid #ffffff;
  outline-offset: 1px;
}
```

**Sizing rationale:** Each `.track-space` uses a 2x2 `grid-template-columns: 1fr 1fr; grid-template-rows: 1fr 1fr` layout to accommodate up to 4 cars per space. At 375px screen width the board is ~359px (100vw - 16px), so each of 6 columns is ~60px, and each car slot is ~30px. 80% of 30px = 24px rendered car — visually clear at pixel art scale.

**Do not** use `border-radius: 50%` on `.car-img` (that was the circle approach — images are rectangular sprites).

**`image-rendering: pixelated`** is mandatory to preserve pixel art crispness when images are scaled down from their ~6MB prototype dimensions.

### Player identification

The player's car (`car-red.png`) is visually distinct by color. Additionally apply `outline: 2px solid #ffffff` via `.car-player` class on the `<img>` element. This mirrors the existing `.car-player` border rule semantics.

### JS Rendering Change in `renderTrack()`

Replace:

```js
var circle = document.createElement("div");
circle.className = "car-circle";
circle.style.background = spaceCars[j].color;
if (spaceCars[j].id === "player") {
  circle.classList.add("car-player");
}
```

With:

```js
var img = document.createElement("img");
img.className = "car-img";
var carIndex = ["player","ai1","ai2","ai3"].indexOf(spaceCars[j].id);
var colorNames = ["red","blue","green","yellow"];
img.src = "images/car-" + colorNames[carIndex] + ".png";
img.alt = spaceCars[j].label;
if (spaceCars[j].id === "player") {
  img.classList.add("car-player");
}
```

## Mobile No-Scroll Layout Contract (D-05)

**Constraint:** During the active race view, the full race UI (header + track board + standings + roll button) must be simultaneously visible on a 375px-wide portrait phone without scrolling. Shop and home views may scroll freely.

### Current body layout (race mode)

```
grid-template-areas:
  "header header"
  "board standings"
  "roll roll"
grid-template-columns: 1fr 200px
grid-template-rows: auto 1fr auto
```

Mobile override at `@media (max-width: 599px)` stacks vertically:

```
grid-template-areas:
  "header"
  "board"
  "standings"
  "roll"
grid-template-columns: 1fr
grid-template-rows: auto auto auto auto
```

### Target layout budget at 375px

The body must use `height: 100dvh` (not `min-height: 100vh`) in race mode to cap the grid to viewport height and prevent scrolling.

| Zone | Budget | Rationale |
|------|--------|-----------|
| Header (`#top-bar`) | 44px | One line of text + 8px padding top/bottom |
| Track board (`#track-board`) | 220px | `min(calc(100vw - 16px), 220px)` — see below |
| Standings panel (`#standings-panel`) | 120px | 4 rows at compact height, 2-column grid |
| Roll area (`#roll-area`) | 72px | Button 44px + 16px padding top + 12px margin |
| Gaps | 24px | 3 × 8px gap between grid rows |
| **Total** | **480px** | Fits within 667px (iPhone SE) and 812px (iPhone X) |

**Track board sizing on mobile:**

```css
@media (max-width: 599px) {
  #track-board {
    width: min(calc(100vw - 16px), 220px);
  }
}
```

This reduces the board from the current `min(100vw - 16px, 480px)` = ~359px at 375px screen down to 220px, freeing ~139px for standings and roll button.

**Standings panel on mobile (already 2-column):**

The existing `grid-template-columns: 1fr 1fr` for `#standings-list` on mobile is correct. Compact the padding:

```css
@media (max-width: 599px) {
  #standings-panel {
    padding: 4px 8px;
  }
  #standings-panel h2 {
    font-size: 12px;
    margin-bottom: 4px;
  }
  .standing-row {
    padding: 4px 0;
  }
  .standings-header {
    font-size: 10px;
  }
  .standing-label {
    font-size: 12px;
  }
  .position-badge {
    font-size: 12px;
  }
}
```

**Body height rule for race mode:**

Apply `height: 100dvh; overflow: hidden;` to `body` only when race view is active. Since views are toggled via JS (`.hidden` class), the executor may add/remove a `body.race-active` class during view transitions:

```css
body.race-active {
  height: 100dvh;
  overflow: hidden;
}
```

Fallback for browsers without `dvh` support: use `height: 100vh`.

**`overflow: hidden` on body must only apply during race view.** Home and shop views retain free scrolling (`body` without `.race-active`).
